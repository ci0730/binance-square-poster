# 币安广场批量发帖工具

面向币安广场创作者的桌面批量发帖工具，基于币安官方 Square OpenAPI，支持多账号运营、AI 辅助写帖、短帖、长文章和带图帖子。

## 功能

- 多账号管理，每个账号可独立配置 API Key、Cookie 与代理
- 批量发布，可设置发帖间隔并实时查看每条结果
- 网络抖动自动重试；发帖确认中断时标记为「待确认」，避免盲目重发
- 自动读取 Windows 系统代理，支持 HTTP / HTTPS / Socks5 与代理连通检测
- AI 生成币安广场风格文案，结合真实行情、资讯、文案风格和防重复策略
- AI 输出截断时自动补全，价格与代币标签会在发布前校正
- 支持纯文本短帖、长文章（带标题 + 封面）
- 支持图片帖（最多 4 张）
- JSON / 文本格式批量导入
- Windows 安装包内置 Chromium，用于拉取历史帖子与互动数据

## 前置要求

1. **Windows 10/11**（推荐直接使用 Release 安装包）
2. **币安广场创作者 API Key**  
   前往 [创作者中心](https://www.binance.com/square/creator-center/home) 创建  
   > API Key 仅用于发帖，不涉及账户资产或交易

## 快速开始

推荐从 GitHub Releases 下载最新的 `币安广场批量发帖-Setup-*.exe` 并安装。

从源码运行需要 Node.js 18+：

```bash
npm install
npm start
```

浏览器打开 **http://localhost:3456**

## 使用步骤

1. 在「账号管理」添加一个或多个币安广场账号，并验证 API Key
2. 在「草稿箱」逐条添加或批量导入内容，也可由 AI 生成草稿
3. 选择发布账号与发帖间隔（建议 3–5 秒）
4. 勾选草稿并发布；若出现「待确认」，先拉取广场历史帖子核对再决定是否重发

## 网络与代理

- 软件优先使用账号独立代理，其次使用全局代理或 Windows 系统代理
- 查询、行情、图片上传等可安全请求会在临时网络错误时自动重试
- 实际发帖请求不会在响应丢失时盲目重发，防止产生重复帖子
- 遇到连接问题时，可在「设置 → 网络代理」检测出网与代理协议

## 导入格式

### JSON

```json
[
  { "text": "Hello #crypto $BTC" },
  { "text": "深度分析正文...", "title": "2026 市场展望" }
]
```

### 文本（每行一条）

```
BTC 看涨 #crypto
深度分析正文|2026 市场展望
```

格式：`内容|标题`，标题可选。

## 限制说明

| 项目 | 限制 |
|------|------|
| 每日发帖 | 100 条 / API Key |
| 每日上传 | 400 次 |
| 短帖图片 | 最多 4 张 |
| 长文章封面 | 1 张 |

## API Key 存储

Key 保存在本地：`~/.config/binance-square/openapi-key`（权限 600）

也可通过环境变量设置：

```bash
set BINANCE_SQUARE_OPENAPI_KEY=your_key_here
npm start
```

## 常见错误

| 错误码 | 说明 |
|--------|------|
| 220003 | API Key 不存在 |
| 220004 | API Key 已过期 |
| 220009 | 今日发帖已达上限 |
| 20002/20022 | 内容含敏感词 |
| 20013 | 内容长度超限 |
